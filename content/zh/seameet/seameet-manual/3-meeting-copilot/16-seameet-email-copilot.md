**簡介**

SeaMeet 是一款由 AI 驅動的會議助理，旨在簡化會議後的工作流程。不同於傳統工具需要使用者將逐字稿複製到外部的大型語言模型 (LLM) 工具（如 ChatGPT 或 Claude），SeaMeet 採用了一種「代理式郵件 (Agentic Email)」工作流程。這讓使用者可以透過直接「回覆」電子郵件串的方式，來生成摘要、草擬信件、建立 GitHub 工單 (Ticket) 以及撰寫工作說明書 (SOW)，完全無需切換分頁或複製貼上內容。

**先決條件**

要開始有效使用 SeaMeet，請確保您已安裝瀏覽器擴充功能：

* **下載：** [SeaMeet Chrome 擴充功能](https://chromewebstore.google.com/detail/seameet-take-chatgpt-meet/gkkhkniggakfgioeeclbllpihmipkcmn?utm_source=SeaMeet&utm_medium=Website&utm_campaign=Landing_Btn_Click&pli=1)
* **帳戶：** 請至 [SeaMeet.ai](https://seameet.ai) 註冊

**代理式工作流程：逐步說明**

**1. 錄製您的會議**
在啟用 SeaMeet 的情況下開始您的會議（透過 Zoom、Google Meet 或 Microsoft Teams）。該工具將即時錄製並轉錄會議內容。

**2. 收到摘要郵件**
會議結束後，SeaMeet Copilot 會自動發送一封電子郵件到您的收件匣。這封郵件包含：

* 討論內容的摘要。
* 建議的行動項目。
* 一個「回覆 (Reply)」提示功能。

**3. 與 Copilot 互動（「回覆」法）**
與其將逐字稿帶到單獨的 AI 工具中，只需點擊摘要郵件的 **回覆 (Reply)** 即可。您可以使用自然語言要求 SeaMeet Copilot 執行特定任務。

* *範例：*「草擬一封給客戶的後續跟進信件，總結接下來的步驟。」
* *範例：*「根據討論的定價建立一份正式的工作說明書 (SOW)。」

**4. 接收生成的內容**
SeaMeet 將處理您的請求，並在同一個電子郵件串中回覆生成的內容。這使得上下文歷史記錄能夠完整保留在一次對話中。

**5. 迭代與修正**
因為上下文保留在電子郵件串中，您可以再次回覆以微調輸出結果。

* *範例：*「讓這封信的語氣更客氣一點。」
* *範例：*「將 SOW 的格式改為列點清單。」

**進階使用案例**

* **專案管理整合：** 您可以要求 SeaMeet 將會議筆記轉換為技術格式。例如：*「為團隊建立一份詳細的 Markdown 格式 GitHub Issue 工單。」* SeaMeet 將生成可直接複製貼上的程式碼區塊。
* **銷售與法律文件：** 透過引用會議錄音中提到的具體細節，生成如 SOW 或提案等複雜文件。
* **脈絡歷史紀錄：** 所有的迭代過程都儲存在電子郵件串中。如果您需要在幾個月後重新檢視為何做出某個決定，完整的 AI 對話歷史紀錄都保存在您的收件匣中。
